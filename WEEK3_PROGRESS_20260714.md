# BNBU Student iOS Week 3 推进记录（2026-07-14）

## 本周目标

本轮按《BNBU Sports Week 3 三端开发任务书》推进，iOS 端负责：

- W3-MOB-01（P0）：7 月 15 日负责人更新口径后，以新增 Android 学生端源码为前端结构基线，对齐导航、页面层级、组件、主题和主要操作路径。

## 2026-07-14 第一轮完成

- 新增真实上传进度：远程凭证上传改为基于 URLSession 实际发送字节回调，不再使用模拟进度。
- 提交页显示当前文件、文件序号、总体百分比、记录提交阶段和结果同步阶段。
- 增加提交防重入状态，同一上传/提交过程未结束时不能再次触发提交。
- 区分安全重试边界：
  - 凭证上传阶段失败，可保留表单并“重试上传”。
  - 记录提交阶段发生网络中断，先刷新历史记录确认结果，避免可能已写入服务器时重复提交。
- 修复服务器已经接受打卡、但随后工作台刷新失败时被前端误判为提交失败的问题：此时保留成功记录并提示稍后同步，不诱导重复打卡。
- 统一常见学生端错误文案：400、401、403、404、408、409、413、422、429、5xx，以及重复提交、日期范围外、文件过大和 Token 失效。
- 保持现有接口路径、请求字段和上传字段 `files` 不变，未擅自新增幂等键或日期字段。

## 2026-07-15 Android 源码对齐

- 已读取新增 Android 工程 `BNBU-Sports-Android-week2/app`，并以 Compose 源码而非文件名错位的旧截图作为前端结构基准。
- 根结构已统一为 `首页 / 课程 / 打卡 / 成绩 / 我的` 五栏，通知改为首页铃铛进入底部弹层，不增加独立通知 Tab。
- Design System 已统一为 18pt 页面边距、16pt 卡片内边距、12pt 卡片圆角、单层中文标题和浅色默认；底栏改为与 Android 一致的胶囊选中态。
- 已将 Android 官方 BNBU 矢量校徽资源转为 iOS Asset，替换旧的临时文字标记。
- 登录页改为单一真实学生登录：移除可见的演示/服务器切换和 API 地址，补齐密码显隐、隐私政策入口及学生端文案。
- 首页删除额外指标宫格、快捷操作区和重复通知列表，保留问候、学时进度、风险、本周行动和近期任务；通知入口位于右上角。
- 课程页统一为当前学期列表 + 可折叠历史课程，并对齐课程卡片的教师、学年、学期、选课状态和详情入口。
- 打卡页从 `任务 / 提交 / 记录` 改为 Android 的 `提交 / 记录`，固定为自主运动打卡；加入 1h/2h 控件、八类运动项目、其他项目输入、每日一次校验、草稿恢复和 `sportType` 服务器字段。
- 打卡记录继续遵守学校最新要求：列表统一只显示“已提交”，不显示待审核、已通过或被驳回；同时补齐图片/视频预览布局。
- 成绩页移除免测入口，保留总分、四项成绩、计算过程、缺失项和来源追溯。
- “我的”页统一为个人资料、申请与审核、我的老师、组织认证与抵扣、设置；免测申请从成绩页迁入，通知从个人页迁入首页。
- 新增耐力跑成绩换算页面，调用 Android 同路径 `/scoring/convert-endurance`，不在 App 内硬编码评分表。
- 补齐 `sportType`、自主打卡草稿、每日一次和耐力跑响应测试；单元测试由 20 项增至 24 项。
- 完成 Android 对齐后的集中 Debug：远程工作台缓存改为“服务器 + 学生账号”双重隔离，避免换账号且服务器异常时误读上一账号数据；退出登录改为先清除本地 Token，再使用旧会话凭据执行服务器注销，消除快速换号竞态。
- UI 回归新增登录隐私政策和耐力跑入口覆盖，串行 5 条主流程全部通过。

## 当前依赖与待确认

- `CourseTask` 当前只有字符串 `deadline`，没有结构化开始时间、结束时间、服务器时间和时区，客户端不能可靠执行日期边界判断。
- 现有提交接口未确认幂等键或请求 ID；iOS 已完成防重入和不确定结果先刷新策略，但跨网络重试的最终幂等仍需服务端支持。
- 上传最大理论组合为 148MB，高于当前 Nginx 120MB；iOS 继续按逐文件上传实现，组合请求限制由服务端容量方案确认。
- `/scoring/convert-endurance` 来自 Android 最新源码，但不在现有 `backend-handoff/openapi.yaml` 中；iOS 已完成页面和解码，仍需测试服确认接口是否已部署。
- Android 源码仍包含免测/打卡“补材料”端点和扩展状态，但现有 iOS 对接手册明确只有 `pending / approved / rejected` 且没有补材料接口；iOS 不复制该过时逻辑，继续采用“驳回后重新申请”。
