# iOS 端周报 - 2026-07-12

## 周目标是什么

本周按照《BNBU Sports Week2 三端开发任务分派》和《BNBU Sports Design System v1.0》推进，iOS 端聚焦三项 P0：

- 完成学生端原生 SwiftUI 界面的视觉风格升级，统一色板、字体、间距和组件规范。
- 完成年级、学年、学期及体测匹配组别的自动推算，并保留历史课程的跨学期查看能力。
- 继续推进测试服联调，补强网络异常、Token 失效、服务器错误和缓存隔离等稳定性处理。

根据本周与学校老师的最新沟通，前端优先级进一步明确：学生打卡记录只需要展示已经提交的历史记录，不再向学生展示待审核、已通过、被驳回、需补材料等审核状态。

## 周完成进度汇报

### 1. 完成 Design System 视觉升级

- 建立 light/dark 自适应语义色，统一 Primary、Secondary、Background、Surface、Surface Variant、Outline、Error 等颜色角色。
- 页面边距统一为 24pt，卡片内边距统一为 18pt，卡片圆角统一为 12pt，主按钮圆角统一为 28pt。
- 字体权重收口为 Regular / Medium，减少过重标题和不一致的文本层级。
- 卡片、输入框、按钮、状态标签、导航栏、TabBar 和分段选择器统一使用设计系统样式。
- 页面主体移除装饰性网格背景，仅登录页保留品牌网格。
- 外观设置收口为“跟随系统 / 浅色模式 / 深色模式”，默认深色，避免继续维护与统一设计规范冲突的多套主题。
- Debug 调试信息只在 Debug 构建中出现，Release 学生端不展示 API Base、Mock、同步队列等开发者内容。

### 2. 完成年份、年级与学期自动推算

- `StudentProfile` 已兼容 `enrollmentYear / admissionYear / entryYear`、`birthDate / birthday`、`gender`、`gradeLevel` 等可能的后端字段名。
- 学年以 9 月 1 日为边界，自动推算当前学年、秋季/春季学期和大一至大四年级。
- 体测区域可根据学生性别、年龄和年级展示匹配组别；体测分数与免测结论仍以服务器返回为准，不在客户端虚构评分阈值。
- 课程按 semester 自动分组，区分当前学期和历史课程，支持跨学期查看。
- 增加跨 9 月 1 日学年边界的单元测试。

### 3. 完成前后端稳定性与异常态处理

- 远程打卡和免测申请调整为等待服务器确认成功后，再清空表单并展示成功提示，避免网络失败时前端误报成功。
- 401 且刷新失败时会清理 Token、退出远程会话，并提示学生重新登录。
- 增加断网、连接失败、网络中断、上传超时、服务器 500 和字段解码变化等学生可理解的错误文案。
- 测试服、生产服和本地演示数据分别缓存，避免不同环境之间串数据。
- 登录成功但工作台暂时加载失败时，可读取最近一次同步数据，并明确展示离线提示。
- 移除服务器返回空数组时自动填充 Mock 课程、任务和认证数据的行为，避免真实数据与演示数据混淆。
- 首页、课程、打卡和成绩页面支持下拉刷新。
- 免测状态继续严格使用 `pending / approved / rejected`，不调用后端不存在的补材料接口。

### 4. 按老师反馈调整学生打卡记录 UI

- 删除打卡记录页的状态筛选，不再展示待审核、已通过、被驳回、需补材料和系统抵扣等选项。
- 学生本人提交的历史打卡记录，在列表和详情中统一显示“已提交”。
- 系统生成的校队/社团抵扣不再混入学生提交记录，仍在学时进度和身份模块中体现。
- 记录详情移除审核时间线、老师反馈、补交材料和重新提交入口，只保留提交时间、时长、凭证和学生说明。
- 首页移除待审核、需补材料统计，改为已提交记录、进行中任务和未读通知。
- 底部打卡 Tab 移除审核数量红色角标，课程卡片不再展示班级待审核数据。
- 提交成功文案统一调整为“已提交”，不再出现“进入老师审核队列”等学生端不需要的信息。
- 后端 `CheckInRecord.status` 字段继续保留用于接口兼容，但不再映射为学生前端的审核标签。

### 5. 完成 Debug 与数据隔离修复

- 修复远程“全部已读”原先未调用真实接口却直接更新 UI 的问题；现在逐条调用通知已读接口，全部成功后才更新本地状态。
- 单条通知已读不再静默忽略网络错误和 Token 错误。
- 修复远程退出后切换演示登录时仍可能显示服务器工作台的问题。
- 远程模式不再展示或执行“重置本地演示数据”，避免 Mock 数据污染服务器缓存。
- 正式学生设置页移除开发者文案，诊断信息仅保留在 Debug 面板。
- 最终检查发现并修复 Release 登录页预填测试凭据的问题：Debug 保留联调便利，Release 默认不填账号和密码，输入框不再使用测试凭据作为占位文案。

## 遇到哪些困难，踩到了哪些坑

- 后端当前没有稳定返回入学年份和出生日期，客户端虽然已完成字段兼容和推算逻辑，但正式数据效果仍依赖后端补齐字段。
- 测试服现有打卡记录和免测申请仍全部处于 pending，暂时无法验证审批后的状态读回与完整跨端数据流。
- 原有学生端沿用了老师审核视角，包含多种审核标签、补材料入口和审核时间线；学校老师确认学生只需看到是否已经提交后，需要同步清理首页、列表、详情、课程卡片、Tab 角标和提示文案，不能只替换一个状态标签。
- 远程通知“全部已读”原实现存在前端先更新、服务器未执行的问题，本周已改为以服务器成功结果为准。
- 演示数据和远程数据原先存在切换后残留的风险，本周完成数据源与缓存隔离，避免联调时误判服务器返回结果。
