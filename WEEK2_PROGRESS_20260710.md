# BNBU Student iOS Week 2 推进记录（2026-07-10）

## 本周目标对齐

本轮严格按《BNBU_Sports_week2_任务》和《BNBU Sports Design System v1.0》推进，只修改原生 SwiftUI 学生端，不新增老师端或管理端功能。

## 已完成

### 1. Design System UI 升级

- 建立 light/dark 自适应语义色：Primary、Secondary、Background、Surface、Surface Variant、Outline、Error 等。
- 按 Quick Reference 收口页面边距 24pt、卡片内边距 18pt、卡片圆角 12pt、主按钮圆角 28pt。
- 页面主体移除网格背景，仅登录页保留品牌网格。
- 全局字体权重收口为 Regular / Medium。
- 卡片、输入框、凭证按钮、状态标签、导航栏和 TabBar 统一使用圆角与语义色。
- 外观设置收口为“跟随系统 / 浅色模式 / 深色模式”，默认深色，避免继续维护多套与负责人设计规范冲突的主题。
- Debug 调试面板只在 Debug 构建出现，正式学生端不展示开发者工具。

### 2. 年份与体测组别自动推算

- `StudentProfile` 新增并兼容解码：`enrollmentYear / admissionYear / entryYear`、`birthDate / birthday`、`gender`、`gradeLevel`。
- 学年边界按 9 月 1 日切换，自动生成当前学年、秋季/春季学期和大一至大四年级。
- 体测区域展示性别、年龄和年级匹配组别；体测分数与免测结果继续以服务器返回为准，客户端不硬编码或虚构评分阈值。
- 课程页按 semester 自动分组，区分当前学期和历史课程。
- 新增 9 月 1 日跨学年单元测试。

### 3. 接口错误与缓存

- 远程打卡、补交、免测改为等待服务器确认成功后再清空表单、弹成功提示，消除“网络失败但 UI 显示成功”。
- 401 刷新失败会清除 token、退出远程会话并提示重新登录。
- 增加断网、超时、网络中断、无法连接、500、字段解码变化的学生端错误文案。
- 远程工作台缓存按测试服/生产服隔离，本地演示工作台与服务器缓存隔离。
- 登录成功但工作台暂时拉取失败时，可读取最近同步数据并明确显示离线提示。
- 移除服务器空数组时回退 Mock 课程、任务、认证的行为。
- 首页、课程、打卡、成绩支持下拉刷新，用于读取最新任务、已提交记录和成绩数据。
- 免测状态和入口严格按现有契约处理：只有 pending / approved / rejected，不调用不存在的补材料接口。

- 修复远程“全部已读”原先调用空实现却直接更新 UI 的问题；现在逐条调用真实通知已读接口，全部成功后才更新本地状态。
