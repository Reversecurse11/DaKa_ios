# Week 3 移动端 UI 对比清单

## 使用说明

2026-07-15 负责人确认学生端前端以新增 Android 工程为对齐基准。核对范围限定为 `BNBU-Sports-Android-week2/app`；同目录的后端和其他项目不属于学生端 UI 基准。已确认不受支持的旧接口逻辑仍以 iOS 对接手册和后端合同为准。

| 对比项 | iOS 当前基线 | Android 验收目标 | 状态 |
|---|---|---|---|
| 底部导航数量 | 5 个 | Android 5 个 | 已对齐 |
| 导航顺序 | 首页 / 课程 / 打卡 / 成绩 / 我的 | 同顺序、同名称 | 已对齐 |
| 导航图标 | 校徽 / book / plus.app / chart / person | 图标语义一致 | 已对齐 |
| 通知入口 | 首页铃铛 + 底部通知弹层 | Android 首页铃铛 + ModalBottomSheet | 已对齐 |
| 安全区 | SwiftUI 原生 TabView / NavigationStack | 底部手势区、刘海和状态栏不遮挡 | iOS 已完成 |
| 页面边距 | 18pt | Android 根页面 18dp | 已对齐 |
| 卡片 | 16pt 内边距、12pt 圆角 | Android Card 16dp / 12dp | 已对齐 |
| 主按钮 | 28pt 圆角、Primary 填充 | 主次操作层级一致 | iOS 已完成 |
| 字体 | 系统字体，仅 Regular / Medium | 标题、正文、标签层级一致 | iOS 已完成 |
| 浅色模式 | M3 light tokens | 无错色、低对比度和系统默认色混入 | iOS 已完成 |
| 深色模式 | #0F172A 背景及对应语义色 | 不使用纯黑替代设计背景 | iOS 已完成 |
| 跟随系统 | AppStorage 持久化并跟随系统 | 切换和重启后行为一致 | iOS 已完成 |
| 首页 | 问候、学时进度、风险、行动计划、近期任务；右上通知 | 与 Android Compose 顺序一致 | 代码已对齐，待同状态截图 |
| 课程 | 当前学期 + 可折叠历史课程、课程详情 | 分组、空状态、详情入口一致 | 代码已对齐，待同状态截图 |
| 打卡 | 提交 / 记录；自主运动、1h/2h、运动项目、凭证 | Android 两段式流程 | 已对齐 |
| 上传进度 | 当前文件 + 文件序号 + 实际总体百分比 | 使用真实上传进度，不模拟成功 | iOS 已完成 |
| 成绩 | 总分、打卡、专项、平时/签到、体测、计算、风险、追溯 | Android GradesScreen 结构 | 已对齐 |
| 通知 | 全部/未读/截止/申请筛选、全部已读、详情 | Android NotificationSheet | 已对齐 |
| 个人中心 | 资料、申请与审核、老师、组织认证、设置 | Android ProfileScreen 结构 | 已对齐 |
| 耐力跑换算 | 800m/1000m、时间输入、服务器结果 | Android EnduranceScoringScreen | UI/请求已完成，待接口部署确认 |
| 空状态 | 说明原因及下一步，不只显示“暂无数据” | 相同业务状态表达相同含义 | iOS 已完成 |
| 加载状态 | 登录、刷新、上传、提交、同步分阶段显示 | 不用单一假加载覆盖所有阶段 | iOS 已完成 |
| Android 三角提示 | iOS 不展示无统一业务含义的三角图标 | Android 按 Week 3 任务移除 | Android 待处理 |
