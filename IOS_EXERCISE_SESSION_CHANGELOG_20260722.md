# iOS 运动会话改造日志（2026-07-22）

## 1. 本次目标

将学生端打卡入口从旧的“手动选择 1h/2h 后直接提交”改造成第一阶段运动会话流程：

> 选择运动类型与项目 → 开始运动 → 实际计时 → 结束运动 → 自动换算学时 → 上传凭证

本次只实现能够在现有工程和 Mock 数据上验证的客户端能力。由于统一的 `API接口规范_OpenAPI.yaml` 仍为空文件，没有编造新的开始运动、结束运动或跨设备检查接口。

## 2. 已完成修改

### 2.1 运动会话领域模型

- 新增课程相关运动、自主其他运动两类运动类别。
- 新增跑步、篮球、足球、羽毛球、游泳、健身、骑行和其他运动项目。
- 新增运动会话的开始时间、结束时间、状态、课程、定位状态及坐标字段。
- “其他运动”名称必填，最多 32 个字符。

### 2.2 实际计时与学时换算

- 不再允许学生手动选择本次学时。
- 不足 1 小时计 0h。
- 满 1 小时、不足 2 小时计 1h。
- 满 2 小时计 2h，并将会话结束时间封顶在开始后 2 小时。
- 手动结束时间不会早于开始时间，也不会超过 2 小时上限。

### 2.3 本地恢复与账号隔离

- 运动开始时先安全持久化，再更新界面状态。
- App 重新进入前台、重新打开或设备重启后，可根据持久化的开始时间恢复经过时长。
- 恢复时校验学生 ID，防止其他账号继承前一账号的运动会话。
- 退出登录时清除运动会话和本地运动日期账本。

### 2.4 每日一次口径

- 新增按运动开始日期记录的本地账本。
- 使用 `Asia/Shanghai` 时区判断运动日期。
- 跨零点运动按开始日期占用每日次数，不再仅依赖提交时间。
- 旧服务器记录没有开始时间时，仍兼容使用提交时间判断。

### 2.5 打卡页状态机

- 未开始：选择运动类别、运动项目并校验当前体育课程。
- 进行中：显示实时经过时间、开始时间、运动项目和定位获取状态。
- 已结束且不足 1 小时：明确提示不计入学时，禁止提交凭证。
- 已结束且达到 1/2 小时：展示自动换算结果，并开放凭证、草稿和提交入口。
- 提交成功后清理已完成会话，并进入打卡记录。

### 2.6 提交后的本地进度

- Mock 提交成功后立即累计课程或自主运动学时。
- 远程提交成功但工作台刷新失败时，使用已确认的提交结果更新本地记录和进度。
- 自主运动原始学时与计入学时分别更新，计入值继续遵守现有上限。

### 2.7 测试支持

- 新增完成 1 小时运动会话的 Debug UI 测试夹具。
- UI 测试不再通过手动选择学时构造提交记录。
- 增加计时边界、2 小时自动结束、本地恢复、跨零点日期及即时进度测试。

## 5. 仍然存在的问题

### P0：进入真实联调前必须解决

1. **统一 OpenAPI 为空。** 无法确认新运动会话端点、请求字段、响应字段、错误码和幂等规则。
2. **尚未接入开始/结束运动 API。** 当前会话由本机保存，服务器不能提供权威开始时间和结束时间。
3. **缺少跨设备活动会话检查。** 同一账号仍可能在另一台设备重复开始运动，必须由服务端唯一约束兜底。
4. **每日一次只有本地增强。** 数据库层仍需按学生 ID、上海时区运动开始日期建立唯一约束。
5. **GPS 尚未实现。** 当前模型和界面支持“定位不可用”降级，但没有申请定位权限或调用 `CoreLocation` 获取开始时单次位置。
6. **后台能力未完成。** 当前使用持久化时间戳恢复并在重新进入前台时结算；尚未配置 `BGTaskScheduler`、后台通知或被系统终止后的服务端补偿机制。
7. **旧任务链路仍存在。** `CourseTask`、任务截止时间、任务列表及 `/student/tasks` 仍用于兼容现有提交接口。
8. **旧审核/补材料链路仍存在。** `pending/approved/rejected/supplement`、驳回、补材料和复审 API 尚未删除。
9. **课程相关运动仍依赖旧任务匹配。** 若当前课程没有可提交的旧任务，完成运动后仍无法提交；需要新接口明确以课程 ID 提交。
10. **记录模型未完成迁移。** 记录页仍缺少运动开始/结束时间、实际秒数、打卡日期、关联课程、有效/无效状态及无效原因。
11. **远程累计学时仍依赖旧工作台合同。** 客户端有刷新失败兜底，但最终权威累计值必须由服务端返回。

### P1：核心链路稳定后补齐

1. 首次进入打卡页的健康安全提醒及按账号记录确认状态。
2. 校队、社团免打卡申请及独立接口。
3. 同类申请 409、防重复申请和最多两次补充材料限制。
4. 教师配置的 0–20h 动态抵扣及课程/自主运动分配规则。
5. 动态学时目标，移除客户端固定 10h + 10h 规则。
6. 新生设置密码、忘记密码、邮件重置及个人设置修改密码。
7. 隐私政策版本、同意时间、首次强制同意和数据删除申请。
8. 中英双语、系统语言自动选择和手动切换。
9. 三步首次引导、离线帮助、服务反馈及工单状态。
10. 成绩申诉、缓考/补考申请和证明材料上传。
11. 成绩空值、缺考、缓考、动态权重及逐项四舍五入模型。
12. APNs 注册、device token 上传、通知去重和点击跳转。
13. App 最低版本检查、强制更新、维护模式及只读模式。
14. 图片原始扩展名/MIME 保真，以及视频时长和批次大小规则确认。

### 发布与环境问题

1. Debug 默认仍连接旧 HTTP 测试地址。
2. Release API 地址仍是 `configuration-required.invalid`。
3. 没有正式 HTTPS 域名、证书及生产签名验收结果。
4. 没有新流程的真实测试账号、稳定测试数据和真机验收证据。
5. 旧 API 审计脚本仍检查任务、审核和补材料合同，不能作为新流程验收依据。
